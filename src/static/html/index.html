Tracker information

See information about the connected trackers. An option with yellow text requires a restart of the connection or program to fully apply.


Program settings

Set the settings of the program. An option with yellow text requires a restart of the program to fully apply.

Tracker model

Select the model of HaritoraX trackers you have, the HaritoraX Wireless or HaritoraX Wired (1.1b/1.1/1.0).

Connection Mode

Select the connection mode used to connect to the trackers. Multiple modes can be used at the same time if wanted.

COM ports

Select the COM ports your GX(6/2) dongles or HaritoraX 1.1b/1.1/1.0 trackers are using. Up to 4 ports can be selected (3 for GX6 and 1 for GX2) unless checks are bypassed.

Update checking

Modify the settings used when checking for updates on program startup.

Other settings

Some settings for the program that don't fit in the other categories.


Tracker settings

Set the global settings for the trackers (can be overriden per-tracker). An option with yellow text requires a restart of the program to fully apply.

FPS transfer rate

Select the FPS transfer rate for the trackers. If using Bluetooth, 50FPS may be more stable.

Sensor mode

Select the sensor mode for the trackers. Mode 1 enables the magnetometer which is best if you have a stable geomagnetic environment (mag status is green or yellow). If unstable (mag status is red), use mode 2.

Sensor auto correction

Enable or disable sensor auto correction features. Usually not needed to be changed and results vary between environments.

About

The SlimeTora project is a middleware program that allows you to connect the HaritoraX trackers to the SlimeVR server.

Debugging

An option with yellow text requires a restart of the program to fully apply.

Enable or disable debugging features for the program. Using logging mode 'All data (!!!)' will cause huge log files and lag, only use if asked or have tracker issues!
